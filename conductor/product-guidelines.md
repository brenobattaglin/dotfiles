# Product Guidelines: dotfiles

## Code Style
- **Minimalist Shell:** Scripts should be simple, readable, and focused. Avoid unnecessary complexity.
- **Self-Documenting Scripts:** Use meaningful names and structure scripts so their purpose is clear without extensive external documentation.

## Commit Messages
- **Conventional Commits:** Use standard conventional commit formats (e.g., `feat:`, `fix:`, `docs:`) to provide clear context for changes.
- **No Conductor Prefixes:** Do not include the word "conductor" or any related prefixes in commit messages.
- **Brief and Concise:** Keep messages short and descriptive.

## Documentation
- **Clear Readability:** Ensure scripts are easy to understand for any user or for your future self.

## Quality and Reliability
- **Static Analysis (Linting):** Use tools like `ShellCheck` to ensure script quality and catch common shell script errors.
- **Consistency:** Maintain the same style and structure across all scripts in the repository.
